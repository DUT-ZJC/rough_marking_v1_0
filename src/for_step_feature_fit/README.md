# STEP 解析面引导的 STL 局部拟合与毛刺标注

## 1. 目标

`src/for_step_feature_fit` 是当前主用的扫描特征拟合流程。

它不再从 STL 三角片里“猜哪里像平面、哪里像柱面”，而是走下面这条路线：

1. 从 STEP 中直接读取已经定义好的解析面类型和修剪范围。
2. 把整个 STEP 模型粗配准到 STL。
3. 在每个 STEP 面附近，只截取一小块 STL 候选区域。
4. 用 STL 三角片重新拟合最终的解析面参数。
5. 用拟合残差把正常区域和毛刺/缺陷/噪声分开。

这套方法的核心原则是：

- STEP 负责“告诉程序去哪里找、找多大范围”。
- STL 负责“给出最终拟合参数是什么”。
- 阈值只负责“筛离群毛刺”，不负责“先判断这个面是不是平面/柱面/曲面”。

## 2. 数据对应关系

当前 `data` 目录中的正确对应关系是：

```text
./data/1_orginal.step <-> ./data/1.stl
./data/2_orginal.step <-> ./data/2.stl
```

建议先用 `2` 号模型调流程，再看 `1` 号模型的大场景效果。

## 3. 运行方法

默认运行：

```powershell
conda activate cgal_env
python -m src.for_step_feature_fit.demo
```

默认会打开：

```text
./data/2_orginal.step + ./data/2.stl
```

运行 `1` 号模型：

```powershell
conda activate cgal_env
python -m src.for_step_feature_fit.demo --step ./data/1_orginal.step --stl ./data/1.stl
```

常用调参方式：

```powershell
conda activate cgal_env
python -m src.for_step_feature_fit.demo `
  --step ./data/2_orginal.step `
  --stl ./data/2.stl `
  --support-gap 2.0 `
  --plane-tol 0.8 `
  --cylinder-tol 0.8 `
  --generic-tol 1.2
```

## 4. 代码结构

- `step_reader.py`
  - 使用 `pythonocc-core` 读取 STEP。
  - 逐个提取 `plane / cylinder / cone / sphere / torus`。
  - 为每个解析面保存修剪面 mesh 和解析参数。

- `pipeline.py`
  - 负责 STEP 与 STL 的整体粗配准。
  - 把 STEP 解析面变换到 STL 坐标系。
  - 为每个面构造局部候选区。
  - 做局部净化、鲁棒最小二乘拟合、全局唯一归属。
  - 输出拟合通过区和毛刺区。

- `view.py`
  - Open3D 图形界面。
  - 左侧按特征类型分组显示列表，并按面积降序排序。
  - 支持鼠标左键直接在 STL 上点选最近特征。
  - 支持实时调整阈值并可视化毛刺。

- `demo.py`
  - 独立测试入口。

- `core_types.py`
  - 会话、阈值、结果等共享数据结构。

## 5. 全流程说明

### 第一步：从 STEP 读取解析面

`step_reader.py` 会直接从 STEP 的 B-Rep / Surface 中读取：

- 平面：点、法向
- 柱面：轴线、半径
- 锥面：顶点、轴线、半角
- 球面：球心、半径
- 环面：中心、轴线、主半径、管半径

这里得到的是 **CAD 原始定义**，不是 STL 三角片阈值判断的结果。

### 第二步：整体配准 STEP 到 STL

因为 STEP 和 STL 坐标系通常不重合，所以不能直接把 STEP 面拿去 STL 上截区域。

`pipeline.py` 会：

1. 把所有 STEP 面合并成一个整体 mesh。
2. 读取 STL mesh。
3. 调用现有粗配准流程，把整个 STEP 变换到 STL 坐标系。
4. 得到每个 STEP 解析面在 STL 坐标系下的位置。

这一步之后，每个 STEP 面都有了“在 STL 里大概应该出现在哪里”的位置。

### 第三步：局部支撑搜索

这里的“局部支撑搜索”就是：

- 围绕当前 STEP 面，在 STL 上找一批“可能属于它”的候选三角片。
- 后面的拟合只在这批三角片里做，不在整个 STL 上做。

当前实现分两层：

1. 先用 **STEP 修剪面三角化后的 AABB** 做粗筛。
2. 再用 **STL 三角片中心到当前 STEP 面 mesh 的最近距离** 做精筛。

其中 `support_gap_mm` 的作用是：

- 控制这个局部搜索带的厚度。
- 值越大，不容易漏，但更容易把邻面和毛刺一起抓进来。
- 值越小，区域更干净，但对配准误差更敏感。

注意：

- 这里的 STEP 面只负责“局部定位”和“候选区搜索”。
- 不负责直接输出最终拟合参数。

### 第四步：局部预处理

候选区截出来以后，会先做本地净化，再进入最终拟合。

当前局部净化包括两类：

- 平面
  - 先用三角法向一致性去掉明显属于邻接平面的区域。
  - 再按连通性只保留最合理的主连通块。

- 柱面 / 锥面 / 球面 / 环面
  - 先用变换后的 STEP 解析面参数做一轮 **宽松的参考残差过滤**。
  - 目标只是删掉明显不属于该面的外来区域。
  - 然后同样只保留最合理的主连通块。

这一步的作用是：

- 不让完全属于别的面的边界三角片先污染当前面的拟合。
- 把“拟合前”就明显错误的候选区先清掉。

### 第五步：用 STL 做最终鲁棒拟合

这是这套流程最关键的部分。

当前五类解析面都已经切到“STEP 给位置与初值，STL 给最终参数”的模式。

每个面当前都采用两段式求解：

1. 先用局部支撑区做一次鲁棒拟合。
2. 再用阈值筛出的内点重新拟合最终参数。

因此最终参数主要由内点决定，不再让明显离群三角片直接参与最终参数计算。

#### 平面

- 输入：局部 STL 三角片中心、法向、面积权重
- 初值：局部主法向和加权中心
- 最终求解：鲁棒最小二乘正交距离拟合

#### 柱面

- 输入：局部 STL 三角片中心、法向、面积权重
- 初值：从 STL 点和法向估计轴向、截面圆心和半径
- 最终求解：鲁棒最小二乘联合优化轴线和半径

#### 锥面

- 输入：局部 STL 三角片中心、面积权重
- 初值：来自 STEP 变换后的顶点、轴线和半角
- 最终求解：鲁棒最小二乘联合优化顶点、轴线和半角

#### 球面

- 输入：局部 STL 三角片中心、面积权重
- 初值：来自 STEP 变换后的球心和半径
- 最终求解：鲁棒最小二乘联合优化球心和半径

#### 环面

- 输入：局部 STL 三角片中心、面积权重
- 初值：来自 STEP 变换后的中心、轴线、主半径、管半径
- 最终求解：鲁棒最小二乘联合优化中心、轴线、主半径和管半径

这里要特别强调：

- STEP 参数只作为 **搜索范围和非线性优化初值**。
- 最终显示出来的解析面参数，以 STL 局部拟合结果为准。
- 这正是为了避免配准误差把 STEP 参数直接带进最终结果。

### 第六步：阈值筛离群毛刺

最终拟合完成后，会对局部支撑区里的每个 STL 三角片计算残差。

- 残差 <= 阈值：认为拟合通过，归入解析面正常区域。
- 残差 > 阈值：认为是毛刺/缺陷/噪声候选。

所以：

- `plane_tol_mm` 只控制平面离群筛选的严格程度。
- `cylinder_tol_mm` 只控制柱面离群筛选的严格程度。
- `generic_tol_mm` 控制锥面、球面、环面的离群筛选。

这些阈值不负责“识别面类型”，只负责“识别离群毛刺”。

### 第七步：全局唯一归属

局部分析完成后，还会做一轮全局冲突消解。

原因是：

- 相邻两个 STEP 面在边界附近，可能同时抓到同一批 STL 三角片。
- 如果不做全局裁决，同一片 STL 三角片可能会同时影响多个面的拟合。

当前实现会：

1. 先对所有面各自跑一遍局部分析。
2. 对所有重叠候选三角片做打分竞争。
3. 每个 STL 三角片最终只分给一个最匹配的面。
4. 用这份“唯一归属”的结果再做一次最终拟合。

这一步的作用是：

- 减少边界互相抢三角片。
- 降低邻面污染对最终参数的影响。

## 6. 鼠标点选机制

当前界面的鼠标点选不是简单地“选最近列表项”，而是：

1. 用相机 `view/projection` 逆矩阵，把鼠标位置转成 3D 射线。
2. 用射线击中 STL 扫描网格，得到命中的三角片 ID 和 3D 命中点。
3. 优先从“包含该 STL 三角片的局部候选区”里找最可能的 STEP 面。
4. 如果没有可靠候选，再回退到“命中点到 STEP 面的最近距离”。

这样做比直接用 `unproject()` 后只按最近面猜测更稳定。

## 7. 界面说明

左侧：

- 按 `平面 / 柱面 / 锥面 / 球面 / 环面` 分组显示。
- 每组内部按 STEP 面积降序排序。
- 可以实时调整：
  - 局部支撑搜索距离
  - 平面拟合容差
  - 柱面拟合容差
  - 其他曲面拟合容差

中间：

- 灰色：原始 STL
- 蓝色：当前 STEP 解析面位置
- 绿色：拟合通过区域
- 橙红色：毛刺/异常区域
- 黄色点：当前特征毛刺三角片中心

右侧：

- 当前特征类型、面积、支撑面积
- 拟合通过面积、毛刺面积
- 残差统计
- 配准信息

## 8. 参数建议

### `support_gap_mm`

适合先固定，再单独调各类容差。

- 偏大：不容易漏，但边界更容易混入邻面。
- 偏小：候选区更干净，但更依赖配准精度。

### `plane_tol_mm`

- 调小：平面更严格，毛刺更容易被标出来。
- 调大：更多 STL 三角片会被当成平面正常区域。

### `cylinder_tol_mm`

- 调小：柱面边界更干净，但更容易把局部粗糙区打成毛刺。
- 调大：柱面更完整，但误收风险增加。

### `generic_tol_mm`

用于锥面、球面、环面。

如果某类曲面局部波动大，可以适当比平面/柱面放宽一些。

## 9. 当前边界

当前版本已经具备完整可用链路，但还存在这些边界：

- 整体质量仍然依赖 STEP 到 STL 的粗配准质量。
- 局部支撑搜索目前仍然是 “AABB 粗筛 + 到 STEP 修剪面 mesh 的最近距离精筛”。
- 如果 STEP 三角化过粗，局部候选区边界会受到影响。
- 当前的全局唯一归属已经能抑制多数边界冲突，但对于极窄边界带，仍可能需要继续加强专门的边界带建模。
